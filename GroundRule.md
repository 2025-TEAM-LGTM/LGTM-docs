## 3. 💻 코드 및 개발 관련 규칙

- **커밋 메시지 규칙**
  - 커밋 메시지 작성 (형식: `작업유형: 작업 내용`)
      - feat: 기능 구현
      - fix: 버그 수정
      - docs: 문서 작업
      - test: test 관련한 코드 추가
      - chore: 코드 수정 없이 설정 변경
      - rename: 파일명, 폴더명을 수정만 한 경우
      - remove: 파일 삭제만 한 경우
  - ex) feat: 회원가입 구현, fix: 로그인 오류 해결
  - 불필요한 커밋은 줄이고, 되도록 **작업 단위로 커밋**

- **코딩 컨벤션**
  - 가능한 한 **주석이나 README에 간단한 설명 남기기**
  - 파일명/함수명은 **직관적인 영어로 작성**
